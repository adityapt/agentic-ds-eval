## Cluster 1: Agentic AI Frameworks and Scaffolding Architectures

### Summary

This cluster covers the design, implementation, and comparative analysis of agent scaffolding systems that wrap large language models to enable multi-step autonomous task execution. The three locked agents in this study—**AIDE** (Weco AI, associated with MLE-bench), **AutoGen** (Microsoft), and **Aider**—each represent distinct architectural philosophies.

- **AIDE** operates via iterative solution trees, using a code-improvement loop scored against a validation metric, making it particularly suited to Kaggle-style ML tasks. Its evaluation on MLE-bench provides the closest existing benchmark to the tasks in this study.
- **AutoGen** implements a conversational multi-agent pattern where specialized sub-agents (e.g., a coder agent and a critic agent) exchange messages in structured dialogues, enabling decomposition of complex pipelines.
- **Aider** is a code-editing agent designed for repository-level interaction, applying diffs to existing codebases via a chat interface, positioning it between pure code-generation and full pipeline orchestration.

**Key tensions** in this cluster include: (a) whether agent scaffolding or base model capability is the dominant performance driver (directly motivating H1), (b) the degree to which agent architectures impose task-specific inductive biases, and (c) reproducibility challenges arising from non-deterministic multi-step execution.

**Methodological analogue:** The MizAR 60 system (jan2023mizar) demonstrates that scaffolded reasoning systems over formal corpora can achieve high task-completion rates, but that the interaction between the underlying prover engine and the scaffolding strategy determines ceiling performance—directly analogous to the agent × base-model interaction tested in H1.

## Cluster 2: LLM Code Generation and Automated Machine Learning (AutoML)

### Summary

This cluster encompasses the growing body of work on LLMs as code generators for data science workflows, and its relationship to classical AutoML systems (Auto-sklearn, TPOT, H2O AutoML, AutoGluon).

- LLM-based code generation has progressed from single-function completion (Codex, 2021) to full notebook generation, with models like DeepSeek-V3 and Qwen2.5-Coder representing the current frontier of open-weight coding models.
- **DeepSeek-V3** is a mixture-of-experts model trained with emphasis on coding and mathematical reasoning, offering competitive performance with proprietary models at open-weight accessibility.
- **Qwen2.5-Coder-32B-Instruct** is Alibaba's instruction-tuned coding specialist, demonstrating strong performance on HumanEval and similar benchmarks.
- **Claude 3.5/4.x Sonnet** (Anthropic) represents the proprietary frontier, with strong instruction-following and extended context capabilities relevant to long data science pipelines.

The distinction between AutoML (which searches over fixed pipelines) and agentic LLM systems (which generate novel pipeline code) is critical: agents can in principle implement domain-specific models (e.g., Robyn's media saturation curves, LightweightMMM's Bayesian priors) that no classical AutoML system would attempt.

**Deep learning review literature** (alzubaidi2021review) contextualizes the architectural evolution from CNNs to transformer-based LLMs, situating the base models in this study within a broader capability trajectory relevant to code generation tasks.
