## Sub-questions

### SQ1 — Variance Attribution: Does Base-Model Choice Dominate Agent-Scaffold Choice in Determining Task Performance?

**Directly addresses:** Pre-registered H1; the central methodological contribution of the 3×3 factorial design.

**Operationalization within locked spec:**
This sub-question is answered by executing the two-way ANOVA on each of the 6 tasks using the 3×3 matrix of 9 agent-model cells, each observed across 5 seeds (n=5 per cell). The ANOVA partitions total performance variance into:
- Main effect A: Agent scaffold (AIDE, AutoGen, Aider) — 2 degrees of freedom
- Main effect B: Base model (Claude 4.6 Sonnet, DeepSeek-V3, Qwen2.5-Coder-32B-Instruct) — 2 degrees of freedom
- Interaction effect A×B — 4 degrees of freedom
- Residual (within-cell seed variance) — 36 degrees of freedom

**H1 rejection criterion (locked):** F-stat for agent ≥ F-stat for model on ≥4 of 6 tasks.

**Metrics per task (locked):**
| Task | Metric |
|---|---|
| MMM | NRMSE vs. Robyn/PyMC-Marketing/LightweightMMM baselines |
| Coupon Redemption | MAP@10 |
| Customer Churn | AUC-ROC |
| Recommendation | MAP@12 |
| Conversion Funnel | AUC-ROC |
| Retail Forecasting | WRMSSE |

**Key literature gap this closes:** MLE-bench (Chan et al., 2024) couples GPT-4o to all agents, making F-stat decomposition impossible. AIDE leaderboard reports single-agent results. No prior work reports a two-way ANOVA decomposing scaffold vs. model variance on any task suite.

**Paper section:** Methodology §3.4 (Statistical Analysis Protocol); Results §4.1 (H1 Test).

### SQ2 — Autonomy Ceiling Asymmetry: Do Attribution-Heavy Tasks (MMM, Forecasting) Impose a Structural Autonomy Ceiling Below L4, While Routine Tasks (Coupon, Churn, Recommendation) Do Not?

**Directly addresses:** Pre-registered H2; the lifecycle × autonomy rubric's primary empirical claim.

**Operationalization within locked spec:**
Each of the 54 agent-task cells (9 agent-model combinations × 6 tasks) is annotated using the 7-stage × 5-level rubric:

**7 Lifecycle Stages:**
1. Problem Framing
2. Data Exploration
3. Feature Engineering
4. Model Selection
5. Validation
6. Interpretation
7. Reporting

**5 Autonomy Levels (SAE-inspired):**
| Level | Label | Description |
|---|---|---|
| L1 | Code Completion | Agent completes code snippets; human directs all decisions |
| L2 | Conditional Automation | Agent executes predefined steps; human monitors |
| L3 | Conditional Autonomy | Agent handles routine sub-tasks; human intervenes at decision points |
| L4 | High Autonomy | Agent completes full pipeline; human reviews outputs only |
| L5 | Full Autonomy | Agent completes, validates, and reports; no human intervention required |

**H2 rejection criterion (locked):** Agents achieve L4 autonomy in ≥60% of cells on Coupon, Churn, Recommendation; AND L3 or below in ≥70% of cells on MMM and Forecasting.

**Annotation protocol:** Two independent raters apply the rubric per stage per cell; inter-rater reliability measured via Cohen's κ. Disagreements resolved by adjudication. Stage-level autonomy scores aggregated to cell-level by taking the minimum autonomy level achieved across all 7 stages (autonomy is bottlenecked by the weakest stage).
